注册中心——服务发现

DiscoveryClient对象（或NacosServiceDiscovery），getServices方法获取所有微服务。getInstances(微服务名称)获得微服务的所有实例

区别：DiscoveryClient 是 Spring Cloud 提供的服务发现客户端接口，而 NacosServiceDiscovery 是其在 Nacos 注册中心中的具体实现类之一。

因为服务发现功能是必备的，所以直接在每个微服务的启动类加上@EnableDiscoveryClient

        相关问题：

        ​ 使用restTemplate来发送请求，第一次会走注册中心，获取所有的微服务的上线实例。然后缓存起来。之后就会缓存这些实例，避免频繁请求注册中心，提高效率。如果服务实例发生变更（下线/上线），定时心跳机制会刷新缓存。

        ​ 注册中心宕机后：

        ​ 缓存了的服务列表仍然有效，只要实例还存活

        ​ 有新的实例上线和下线也无法感知了

配置中心——动态刷新

springboot的配置批量绑定，无需@RefreshScope就能实现动态刷新，短横线和下划线都能映射成驼峰式

  • @Data是必须的，注入需要set方法
  • 加上@Component后才能使用@ConfigurationProperties("")
  • 可以实现自动刷新
  • 可以实现短横线和下划线的映射

OpenFeign

用OpenFeign代替RestTemplate，能自己做负载均衡

注意：Spring Cloud Feign 会校验 @FeignClient 的 value 或 name 字段，必须是一个 合法的服务名称，而服务名称最终会被用作服务发现中的主机名，因此 不能包含下划线 _。
