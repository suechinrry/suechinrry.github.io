服务发现

DiscoveryClient对象（或NacosServiceDiscovery），getServices方法获取所有微服务。getInstances(微服务名称)获得改微服务的所有实例

区别：DiscoveryClient 是 Spring Cloud 提供的服务发现客户端接口，而 NacosServiceDiscovery 是其在 Nacos 注册中心中的具体实现类之一。

因为服务发现功能是必备的，所以直接在每个微服务的启动类加上@EnableDiscoveryClient

远程调用

  • 对于每个微服务的pojo对象，某个微服务可能会用到另一个的pojo。所以可以单独抽取一层（例如命名为model），来存放所有微服务的数据模型。model层和service同级别，使用时当作依赖引入每个微服务的pom.xml

      用 @Bean 手动注入RestTemplate

      在你的配置类中（一般是 @SpringBootApplication 或 @Configuration 的类）加上：

      @Configuration
public class RestTemplateConfig {

    @Bean
    public RestTemplate restTemplate() {
        return new RestTemplate();
    }
}
