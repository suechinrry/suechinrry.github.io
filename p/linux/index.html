Linux相关

linux的目录结构

/为根路径。 例如： /user/test/hello.txt

命令和命令行

命令：一个命令就是一个linux程序，没有图形化界面，可以在命令行提供字符化反馈

命令行：linux终端，是一种命令提示符页面

命令格式

在 Linux 中，命令的一般格式如下：

1

command [options] [arguments]
  1. command：要执行的命令，比如 ls、cd、cp 等。
  2. options（可选项）：修改命令行为的选项，通常以一个或两个连字符（- 或 --）开头。例如：-l、--help。
  3. arguments（参数，可选项）：命令操作的对象，通常是文件名、目录名或者其他需要命令处理的内容

Home目录和工作目录

Home目录：每个用户通常都会在 /home 目录下有一个以用户名命名的子目录，作为该用户的主目录。 例如：==/home/用户名==

工作目录：命令行在执行命令时，需要一个工作目录，默认设置工作目录在该用户的home目录

特殊路径符

  • . 表示当前目录，例如：cd ./desktop等同于 cd desktop
  • .. 表示上级目录，例如：cd ..就是返回上一级，cd ../..就是返回上两级
  • ~ 表示home目录，例如：cd ~等同于cd

命令

ls命令 

1

ls [-a -l -h] [linux路径]

作用：列出当前目录的内容，默认为home目录

  • -a：-all，列出全部文件夹，包括隐藏的文件/文件夹（==文件或文件夹以 . 开头就能自动隐藏==）

  • -l：以列表（竖向）展示内容，并且展示更多的信息。包含==权限信息，所属用户，所属用户组==

  • -h：可读格式显示文件大小

  • ==混合使用==：

    例如：同时使用-a和-l

    • 写法一：-a -l
    • 写法二：-al，-la

创建目录命令

mkdir创建目录

只有在home目录下才有创建目录的权限

1

mkdir [-p] [linux路径]

创建新的目录

  • 不带选项：创建目录时要求父目录必须存在。如果父目录不存在，则会报错。
  • -p：用于创建多级目录，如果某个中间目录不存在，-p 会自动创建它们，而不会报错。例如：
1

mkdir -p /home/用户名/project/src

如果 /home/user/project/ 和 /home/user/project/src 目录不存在，mkdir -p 会依次创建它们。如果目录已经存在，它也不会报错。

  • mkdir 只会创建目录，不会切换到新目录。

在ls -l命令下的==以d开头的是文件夹，以-开头的是文件==。

  • 参数1：是被移动的文件（文件夹）

rm删除文件或文件夹 
1

rm [-r -f] 参数1 参数2 ... ... 参数n
  • -r：和cp一样，-r用于操作文件夹
  • -f：强制删除，强制删除不会有提示。普通用户删除内容不会有提示，root用户删除内容时才有提示。（普通用户一般用不到-f）。
  • 参数1 参数2 … … 参数n，参数代表要删除的文件或路径，用空格隔开。

find查找文件所在位置

  • 按文件名查找

    1

    find 起始路径 -name 文件名
    • 有空格或特殊字符时，打引号，“文件名”

    例如，在/路径下查找文件名为aaa的所在位置， find / -name “aaa”

  • 按文件大小查找

    1

    find 起始路径 -size +(或-)n[kMG]

wc统计文件内容 
1

  • 不使用任何选项，wc 命令会默认输出 行数、单词数和字节数 三项统计信息。

systemctl命令控制软件的关闭和启动 

1

systemctl [start | stop | restart | reload | status | enable | disable] 服务名称

  • start：启动某个服务。

  • stop：停止某个服务。

端口

通过端口可以锁定计算机的具体程序

Linux 系统中，端口是标识网络服务的通信通道，共有 65535 个端口号可供使用

  • 0 - 1023：知名端口（Well-known Ports），由 IANA（Internet Assigned Numbers Authority）分配，用于重要的服务和应用，例如 HTTP（80）、HTTPS（443）、SSH（22）、FTP（21）等。

  • 1024 - 49151：注册端口（Registered Ports），这些端口号是为某些应用程序、服务和进程保留的，但不如知名端口那样严格，通常由开发者或组织申请使用。通常可以随意使用

  • 49152 - 65535：动态端口（Dynamic or Private Ports），这些端口通常用于客户端的临时端口，如在建立连接时操作系统会自动分配这些端口。

基本选项

  • -d <时间间隔>：设置刷新间隔（单位为秒）。默认值通常为 3 秒，例如 top -d 2 表示每 2 秒刷新一次。
  • -n <次数>：指定刷新次数后退出。例如，top -n 5 表示刷新 5 次后退出。
  • -b：以批处理模式运行，输出结果适合重定向到文件。例如，top -b -n 1 > output.txt 将当前信息保存到 output.txt。
  • -p <PID>：仅显示指定 PID 的进程信息。例如，top -p 1234 只监控进程号为 1234 的进程。
  • -u <用户名>：仅显示指定用户的进程。例如，top -u root。
  • -U <用户ID>：与 -u 类似，但通过用户 ID 过滤。
  • -H：显示线程级别的信息。
  • -o <字段>：按指定字段排序。例如，top -o %CPU 按 CPU 使用率排序。

运行时交互键

当运行 top 后，可以通过以下键盘操作动态调整显示内容：

  • 排序相关
    • P：按 CPU 使用率排序（默认）。
    • M：按内存使用量排序。
    • T：按进程运行时间排序。
  • 筛选相关
    • u：筛选特定用户的进程。
    • k：终止指定 PID 的进程。
  • 显示相关
    • z：切换颜色显示。
    • x：高亮排序列。
    • c：切换显示完整命令行或仅显示进程名称。
    • f：调整显示字段。
    • h：显示帮助信息。
    • q：退出 top。

压缩和解压

tar和gzip更为常用

.tar

  • 定义： .tar 是一种打包格式，用于将多个文件和目录合并成一个文件，方便归档和传输。
  • 特点
    • 仅打包文件和目录，不会压缩数据。
    • 文件大小保持不变。
    • 通常用于备份和归档。

.gz

  • 定义： .gz 是一种压缩格式，基于 Gzip（GNU zip）压缩算法，用于减少文件大小。
  • 特点
    • 只能压缩单个文件。
    • 不具备打包功能。
    • 常用于网络传输或磁盘存储优化。

.tar.gz

  • 定义： .tar.gz 是 .tar 和 .gz 的组合，先将文件和目录打包成 .tar 文件，再对 .tar 文件进行 Gzip 压缩。
  • 特点
    • 兼具打包和压缩功能。
    • 是 Linux 系统中最常见的压缩包格式。

tar命令压缩或解压tar和gzip文件 

1

tar [-z -c -x -C -v -f] 参数1 参数2 ... ... 参数n
  • -z：不管是解压还是压缩，都使用gzip算法。不写则使用tarball算法，要使用的话一般放在第一个
  • -c：创建压缩文件，用于压缩模式
  • -x：解压模式
  • -C：用于解压模式，选择解压目的地
  • -v：查看进度。用于显示压缩、解压过程
  • -f：指定打包或解压的文件名（如 archive.tar）。必须放在所有选项的最后一个
常用组合
压缩

  • -cvf 将若干文件压缩到某个.tar文件

    例如，将1.txt和2.txt压缩到test.tar中

    1

    tar -cvf test.tar 1.txt 2.txt

    该命令会将当前目录下的 1.txt 和 2.txt 文件打包到一个名为 test.tar 的归档文件中。

    打包完成后，1.txt 和 2.txt 原文件依然保留，它们的副本被存储在 test.tar 中。

  • -zcvf

    先将 1.txt 和 2.txt 文件打包为一个 test.tar 文件，然后用 gzip 压缩生成最终的 test.tar.gz 文件。

    1

    tar -zcvf test.tar 1.txt 2.txt
解压

  • -xvf 解压到当前目录。

    1

    tar -xvf test.tar

  • -xvf 解压到指定目录。

    1

    tar -xvf test.tar -C /home/jieya

  • -zxvf 解压到指定目录。

    1

    tar -zxvf test.tar.gz -C /home/jieya
